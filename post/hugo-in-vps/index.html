Hugo in VPS

@Can Yellow · Nov 4, 2018 · 2 min read

在Vultr上申请了了一个VPS，本来一直只用来科学上网用的，最近心血来潮打算好好利用VPS，在在上面利用Hugo搭建一个个人静态博客。本来以为比较简单的，没想到折腾了两天，特地把建立过程与踩的坑记录下来，正好贡献新博客的处女作。

第一步：注册建立VPS

因为是在已经购买的VPS上建站，这一步我不需要做。为了完整性，再整理一遍。

注册账户

到Vultr的官网注册账号，官网网址： Vultr官网

截止撰写本文的时候，官网页面是附图vultr官网这样的，还是很简洁大气的!

购买VPS

  1. 注册完毕，直接使用账号密码登录vultr，就可以购买VPS服务了。
  2. 顺利登录后，页面自动跳转到账户控制台，点击server选项，在页面右侧选择蓝色的加号（显示提示：deploy new server）。
  3. 点击后，系统会跳转到Deploy New Instance（部署新的安装）页面，在该页面下按顺序依次选择：
    • Server Location:Singapore. 服务器地址没有特殊要求，我个人偏好选择新加坡(Singapore)。
    • Server Type:CentOS. VPS的操作系统我选的是CentOS7，如果后面重新选的话，我也会考虑改成Ubuntu的，毕竟我本地自用的是Ubuntu系统。当然要选Windows系统也不是不可以，不过安装完成估计十多GB的空间就没有了。
    • Server Size:20GB SSD. 个人使用20GB SSD,512内存,500GB带宽够用了，除了更低的只能走IPv6流量的2.5$/月的套餐，这是最经济的了。
    • Additional Features:可以勾选Enable IPv6. 不许用在意，默认就好，虽然我勾选了IPv6选项
    • Startup Script：留空. 系统启动时执行的脚本，保持默认的留空状态即可，即使有需求，这一项，也可以安装好了后在系统里面更改配置文件的。
    • SSH Keys：输入用其他软件生成的SSH密钥. 密钥的生成方式请参考vultr:How Do I Generate SSH Keys？。使用SSH的意义在于，VPS部署好了后，在本地使用Putty等远程软件登录VPS的命令行时，可以不用输入原来的用户密码，只需要输入一个生成SSH密钥时我们提供的字段（passphrase）就可以登录。如果这一项部署时没有输入，也可以在系统安装完成后，通过系统的命令行操作添加。
    • Server Hostname&Label:你希望的服务器名称与标签类别,无特殊要求.
  4. 下面根据你购买的套餐支付费用即可。Vultr现在可以用支付宝支付了，所以这一步现在应该没有什么难度了。

整个过程的图示，见附图购买VPS

第二步：安装Hugo

尝试安装预编译二进制文件（binary）

登录Hugo官网预编译文件下载页Hugo预编译安装包下载，下载系统对应的预编译的二进制文件。对于Debian，ubuntu，windows，MacOS，官网都能找到现成的预编译的二进制安装包。亲测win7，ubuntu可用。

第一个大坑：官网没有CentOS对应的安装包

由于我的VPS是CentOS7的，官网没有现成的预编译二进制包。抱着试试看的想法，我打算尝试使用CentOS7默认的包管理器yum安装看看是不是在线的软件仓库里有现成的Hugo包可以安装。

$ sudo yum install hugo
> no package hugo available
> Nothing to do

结果并没有安装成功，因为在线的包管理仓库中并没有hugo的安装包。经过万能的google，在Hugo的安装指南中找到了CentOS的解决方案，在其中的CentOS系统安装方法中有如下说明：

Fedora, Red Hat and CentOS

Fedora maintains an official package for Hugo which may be installed with: >sudo dnf install hugo

For the latest version, the Hugo package maintained by @daftaupe at Fedora Copr is recommended: >https://copr.fedorainfracloud.org/coprs/daftaupe/hugo/

See the related discussion in the Hugo forums.

所以，我按照上面提到的方法在命令行中输入安装命令，却收到了没有dnf命令的错误提示。

$ sudo dnf install hugo
> dnf: command not found

显然CentOS系里，默认的包管理器是yum。dnf是新一代的包管理器，fedora22以上的版本已经内置了，但是RHEL与CentOS还是用yum进行包管理。找不到dnf说明没有安装该软件，我本可以先安装dnf软件的，再继续上面的过程就好了。具体的从yum安装dnf过程，参考DNF的命令使用教学，说明的很详细。不过当时我自己走进了死胡同，我想：我使用yum安装不了，说明yum里面没有将上面的软件源添加进去，所以我把上面的含有Hugo的软件源添加到yum中，应该也能安装，于是就有了下面的命令，具体的教程参见How to install Hugo on Linux：

$ sudo vi /etc/yum.repos.d/CentOS-hugo.repo

在上面命令新建的repo源文件中按键盘上的i进入编辑模式，中键粘贴如下内容

[daftaupe-hugo]
name=Copr repo for hugo owned by daftaupe
baseurl=https://copr-be.cloud.fedoraproject.org/results/daftaupe/hugo/epel-7-$basearch/
type=rpm-md
skip_if_unavailable=True
gpgcheck=1
gpgkey=https://copr-be.cloud.fedoraproject.org/results/daftaupe/hugo/pubkey.gpg
repo_gpgcheck=0
enabled=1
enabled_metadata=1

如此便将Hugo的源加入了yum的默认软件源中，使用以下命令安装Hugo。

$ sudo yum update //更新源数据
$ sudo yum install -y hugo   //安装hugo

安装完成后，使用以下命令确认Hugo是否安装OK。出现版本信息，说明安装完成。

$ hugo version
> Hugo Static Site Generator v0.50/extended linux/amd64 BuildDate: 2018-10-29T10:09:56Z

第二个大坑：预编译版本不支持sass

安装完成后，我建立博客文件夹，并使用了hugo-nuo主题，用hugo生成时报错，具体的错误信息没有存档，大概与如下内容类似：

error: failed to transform resource: failed to transform "scss/main.css" (text/css): this feature is not available in your current Hugo version

经过google，确认问题在于我们安装的预编译的Hugo包不支持sass/scss功能，所谓的sass/scss可以简单的理解成更高级的css样式文件，里面能够添加变量、嵌套等功能，但是问题是Hugo默认的预编译包是不包含这个功能的。要使用该功能，只有两条路，一是回到Hugo预编译安装包下载页面下载文件名中带有extended字样的对应的安装包，而是从源代码编译二进制文件，编译时选择extended选项。问题的关键是，跟上面的问题一样，Hugo官方的预编译的extended版本只有win版与ubuntu版的deb包。我们如果想要在CentOS上使用extended版，只剩下一条路，那就是从头编译Hugo了。

注意：之所以报错，是因为我选的主题里面用到了sass功能，必须Hugo支持。如果你的主题没有使用sass功能，安装正常的Hugo就可以了，也就是说第二步到这里就已经完成了。

尝试从源代码编译安装

安装go语言环境

Hugo是基于go语言开发的，因此编译Hugo要先准备go语言环境，下面的命令安装go语言的环境：

$ sudo yum install -y golang

编译Hugo出错

go语言环境安装OK后，就可以在此基础上编译安装Hugo了。但是Hugo项目主页中的编译方法实际上会报错，里面提到的编译方法如下：

mkdir $HOME/src
cd $HOME/src
git clone https://github.com/gohugoio/hugo.git
cd hugo
go install

最终，会报错，报错内容我没有存留，大致是”the package “github.com/gohugoio/hugo” is not found in home/sername/src/github.com/gohugoio/hugo”。在我这里报错是两个原因：

  • git下载的hugo源码的存放目录错误。go语言新版安装时会在环境变量里建立一个\$GOPATH的变量，该变量通常是”\$home$/go”，在我的VPS上对应于”home/username/go”。如果不知道，可以在终端里输入以下命令查询 $ go env

  • install命令不能自动下载并安装Hugo编译所需的依赖的各种库。所以即使源码放置的位置正确也会出错。

    编译Hugo的正确打开方式

    所以基于go语言的Hugo的正确打开方式是使用get命令。因为get命令可以自动安装所需的依赖。

    $ go get -D github.com/gohugoio/hugo      
//下载Hugo，-D表示下载完毕但是不安装，使用get可以下载所有依赖
//编译Hugo-extended版需要sass库的支持，所以还要安装libsass库
$ go get github.com/wellington/go-libsass
//安装go-libsass后，可以安装Hugo了,进入Hugo目录
$ cd home/username/go/src/github.com/gohugoio/hugo
$ go install --tags extended
//--tags extended表示安装extended版，即添加sass支持，这个功能需要上面go-libsass库支持

    编译Hugo后的使用

    安装预编译的Hugo，我们可以在命令行直接运行，win版可以将Hugo的执行文件放到当前目录，然后在当前目录打开命令行运行Hugo命令。但是编译运行稍有不同，编译好后我们需要简单的配置一下。 go语言编译好的可执行文件存放在”home/username/go/bin”目录中，可以在这个目录使用”./hugo”命令在终端运行Hugo。

    $ cd home/username/go/bin/
$ .hugo version

    显然不能这样运行Hugo，太麻烦了，我们可以使用alias命令创建一个hugo的别名

    $ alias hugo='home/username/go/bin/hugo'
$ hugo version  //确认是否可用

    但是这样只在当前的终端中有效。所以需要把上面的alias命令写入系统，在每次终端打开时都能使用。Linux中终端打开时会首先运行用户目录中的”./bashrc”文件。所以可以把上面的alias命令写入”./bashrc”中。

    sudo vi ./bashrc

    然后将”alias hugo=‘home/username/go/bin/hugo’“写入”./bashrc”文件末尾，esc退出vi的编辑模式，按下:wq保存并退出。则hugo命令在命令行默认可用了。

    第三个大坑：go-libsass安装出错

    安装go-libsass时会提示g++:enpty declaration。这里是stackoverflow上关于go-libsass安装enpty declaration错误的解决方案 >With the attribute warning: empty declaration will be returned on older GCC versions (< 7.x)

原因:CentOS上面的GCC套件版本是4.4.7。C++的attribute特性只有GCC套件的版本>7.0.0才提供支持，所以只能重新编译安装较新的GCC7.3.0安装。

第三步：配置Hugo

在完成上面Hugo的安装或者在其他系统上安装完成Hugo，具体以在命令行运行”hugo version”可以输出Hugo的版本信息为准，说明安装OK。

hugo new site 建立网站架构

$ hugo new site filename    
//在当前目录建立以filename命令的网站文件夹

hugo new [post/filename.md] 建立日志文件

$ hugo new post/about.md
//在网站的content/post文件夹里建立about.md的文件，用于博客文章的源文件

安装主题

在当前目录是网站文件夹的情况下，cd进入themes目录。

$ cd themes
$ git clone https://github.com/laozhu/hugo-nuo

Hugo server 启动服务器

go server   //默认设置生成网站，只能本机用localhost:1313访问

我们后面将DNS与域名关联解析完成后，就可以使用下面的命令重新生成网站了，到时候同时还要编辑网站文件夹中的congfig.toml文件，将baseURL改成我们的域名+1313端口的格式，后面再继续这个话题。
