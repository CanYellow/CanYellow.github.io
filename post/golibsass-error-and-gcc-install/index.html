Golibsass Error and Gcc Install

@Can Yellow · Nov 12, 2018 · 2 min read

上次说到在我的VPS上因为gcc编译器的版本太低（gcc4.4.7），导致go-libsass安装出错，提示empty declaration，需要升级gcc到7.0.0以上的版本才能正常编译gcc库。

原因补充说明

关于go-libsass报错的原因，这里有详细的说明可以参考。

大致就是'__attribute__ ((fallthrough))'特性是GCC7新加进去的，与老版本是不兼容的，为此，如果在小于7.0的版本的GCC上运行上面的代码，就会得到我们上面的报错，编译器会认为变量没有声明。
解决方案就是升级GCC编译器到7.0以上。

升级gcc编译器

gcc编译器官网并没有给出打包好的安装文件，因此我们只能从源代码编译生成新版gcc编译器所需的可执行文件、库文件以及头文件。幸好，gcc官网还是给出了一个编译指南的，在这里。除此之外，我在知乎上也找到了一份gcc7.2.0的编译指南。以下是编译过程的记录。

准备工作，安装工具包

这一步，我不需要做，因为我的系统上已经安装好了，只是原有的gcc版本过低而已。对于没有gcc等基础工具的人，需要执行以下命令安装一整套工具。

$ sudo yum  groupinstall Development tools

注意： 本文中所有的代码块，起首的$号表示终端输入的命令，起首的>号表示终端交互显示的内容，符号本身都不需要输入。

如果，你无法确定你的电脑上有没有安装一整套的工具，可以在终端里面输入下面的代码：

$ gcc --version

如果显示类似如下的版本信息的话，说明已经安装好了。

gcc (Ubuntu 7.3.0-27ubuntu1~18.04) 7.3.0
Copyright (C) 2017 Free Software Foundation, Inc.
This is free software; see the source for copying conditions.  There is NO
warranty; not even for MERCHANTABILITY or FITNESS FOR A PARTICULAR PURPOSE.

如果显示command not found，说明尚未安装，那就需要执行上面的安装命令了。

下载源代码

在终端里面输入下面的代码下载源代码

$ mkdir ~/src
$ cd ~/src
$ wget https://mirrors.ustc.edu.cn/gnu/gcc/gcc-7.3.0/gcc-7.3.0.tar.xz

注意：我打算将下载的压缩文件放到我新建的文件夹~/src中,~表示在我的主目录下面，这个路径不是绝对的，请自行更改，倒但是更改后，后面碰到对应的路径，请做对应的变更。

解压源代码

tar.xz格式的文件是经过两层压缩的，所以解压缩需要分步执行。

$ xz -d gcc-7.3.0.tar.xz
$ tar xvf gcc-7.3.0.tar

此时在当前目录应该能够看到解压出来的gcc-7.3.0文件夹了

$ ls ~/src
> gcc-7.3.0 gcc-7.3.0.tar.xz

添加依赖关系

源码文件夹中的download_prerequisites文件指定了安装所需的依赖包，因此我们要首先运行一下该文件把需要的依赖先安装好。

$ ./contrib/download_prerequisites

如果提示command not found一类的错误，就先改权限再执行，换成下面的代码：

$ sudo chmod  755 ./contrib/download_prerequisites
$ ./contrib/download_prerequisites

建立内存缓存

由于我的VPS内存只有512MB，第一次编译过程中由于内存不足，直接报错中断了编译进程，所以第一次编译以失败告终。也就是说在编译之前先要确认内存是否够用，我的内存不足，只能堆内存页面文件了（也就是用硬盘存储空间代替内存），我没有尝试过，保守估计需要2GB内存，不过4GB保险些。

这里网上的出错信息，当时的出错信息与之类似，由于内存不足，第四个进程直接退出了

g++: internal compiler error: Killed (program cc1plus)

先确认有无内存页面文件

$ free -m

如果结果中有显示swap的值是0，说明没有内存页面缓存，我们需要新增内存页面文件，步骤如下：

$ dd if=/dev/zero of=/swapfile count=4096 bs=1M
$ ls / | grep swapfile
$ mkswap /swapfile
$ sudo gedit /etc/fstab

在最后一行命令执行后，会使用gedit打开fstab文件，在文件里新增如下内容：

/swapfile   none    swap    sw    0   0

现在我们重启系统，内存页面文件就会自动加载了。

当然，如果你的系统实际内存超过4GB，那你就不需要建立内存页面缓存了，这一步对你是多余的。

编译安装

$ cd ~
$ mkdir gcc
$ cd ~/src/gcc-7.3.0
$ ../configure --prefix=~/gcc/ --enable-checking=release 
--enable-languages=c,c++ --disable-multilib

上面的代码中，首先在刚刚的新建文件夹gcc下存放编译后的文件，所以将–prefix定义为~/gcc/。上面的最后一行代码是提供给gcc的编译选项。编译准备已经完成。

下面是正式的编译安装了，时间较长，在我的VPS上执行了超过4个小时。

$ make j4
$ sudo make install

建立新版本链接

由于虽然已经编译完成，但是我将编译好的文件配置在~/gcc目录中，编译好后，目录结构如下：

$ ls ~/gcc
> bin  include  lib  lib64  libexec  share

现在需要将新版本的编译好的文件加入系统的环境变量 对应的路径中，保证系统能默认找到gcc新版本，gcc老版本我直接屏蔽了。 所以先将gcc的可执行文件备份，再替换成新的文件。

$ sudo mv /usr/bin/gcc /usr/bin/gcc-4.4.7   //备份gcc
$ sudo mv /usr/bin/gcc /usr/bin/g++-4.4.7   //备份g++
$ sudo cp ~/gcc/bin/g++ /usr/bin/g++
$ sudo cp ~/gcc/bin/gcc /usr/bin/gcc

include的头文件没有变化，所以不需要替换，但是lib64里面的库需要更新

$ sudo mv /usr/lib64/ /usr/lib64-4.4.7   //备份g++
$ sudo ln -s ~/gcc/lib64 /usr/lib64     //建立库文件的软连接

因为库文件里也就libstdc++有变，此处也可以

ln -s ~/gcc/lib64/libstdc++.so.6.0.24 /usr/lib64/libstdc++.so.6.0.24

查看版本，确认是否安装成功

与最初的命令一样，查看下gcc的当前版本，确认是不是7.3.0，输入下面的命令，如果输出gcc版本为7.3.0说明编译成功

$ gcc --version

  • Gcc7.3.0
